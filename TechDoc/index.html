Introduction

A Geographic Information System (GIS) is computer software that allows users to collect, store, maintain, manipulate, and analyze spatial data. QGIS is one example software. It is one of the most widely used GIS platforms, partly because it is an open-source software project. The software is built from a community-driven, collaborative efforts. It is free, stable, and changes through time as different people join the development community.

One of the most powerful facets of a GIS is its ability to automate tasks. Whether one wants to clip 100 different layers or count all the points within the boundaries of a county, a good GIS will make these tasks much easier and faster. While GIS have built-in functions to do things like count points, sometimes the task needs a more user-specific automation algorithm. That is where scripting comes into play, where a GIS pairs with some high-level scripting language to provide user automation capabilities. QGIS provides python scripting functionality for these purposes.

QGIS and Python

The Python Console

Like the terminal on your Operating System, the Python console in QGIS allows you to speak directly with computer. The language you must speak, of course, is Python. The Python Console can be opened by selecting Plugins -> Python Console from the drop down menu at the top of the program. It should look similar to this:

It is important to note that the following code is automatically executed when the console is started. 

			
				from qgis.core import*
				import qgis.utils

This code imports all of the functions in the qgis.core library. However, if you are developing scripts outside of the Python Console, these statements must be called to acquire the same functionality as the built in console (your computer doesn't automatically know QGIS functions).

Standalone Python Scripts

The QGIS python documentation provides this example standalone script with the accompanying description: 

			
				from qgis.core import *

				# supply path to qgis install location
				QgsApplication.setPrefixPath("/path/to/qgis/installation", True)

				# create a reference to the QgsApplication, setting the
				# second argument to False disables the GUI
				qgs = QgsApplication([], False)

				# load providers
				qgs.initQgis()

				# Write your code here to load some layers, use processing algorithms, etc.

				# When your script is complete, call exitQgis() to remove the provider and
				# layer registries from memory
				qgs.exitQgis()

We begin by importing the qgis.core module and then configuring the prefix path. The prefix path is the location where QGIS is installed on your system. It is configured in the script by calling the setPrefixPath method. The second argument of setPrefixPath is set to True, which controls whether the default paths are used.

The QGIS install path varies by platform; the easiest way to find it for your your system is to use the Python Console from within QGIS and look at the output from running QgsApplication.prefixPath().

After the prefix path is configured, we save a reference to QgsApplication in the variable qgs. The second argument is set to False, which indicates that we do not plan to use the GUI since we are writing a standalone script. With the QgsApplication configured, we load the QGIS data providers and layer registry by calling the qgs.initQgis() method. With QGIS initialized, we are ready to write the rest of the script. Finally, we wrap up by calling qgs.exitQgis() to remove the data providers and layer registry from memory.

Concluding Remarks

This document only scratches the surface of how to use Python with QGIS. There is a lot more to learn. I recommend practicing running some simple geoprocessing tools in the Python Console to begin learning the Qgs syntax. From there, you should be able to start writing your own standalone Qgs scripts and maybe even an API.
